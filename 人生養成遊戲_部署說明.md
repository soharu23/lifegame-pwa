# 人生養成遊戲 PWA 部署說明

這個資料夾是「人生養成遊戲」的手機網頁 App 版本：

`life-rpg-pwa-zeabur`

它是一個純靜態 PWA 網站，可以部署到 GitHub Pages，並用手機瀏覽器開啟、加入主畫面。

## 主要功能

- 任務打卡
- 角色頁面，可填寫名字、星座、年齡、tag、自由描述與理想小目標
- 角色頁中的編輯區塊可收合，理想小目標可用清單呈現
- 背包內容整合進角色頁
- 完成任務時有提示音與完成狀態
- 自動累積技能 EXP
- 自動判斷技能等級
- 本周任務板可自由增減顯示任務
- 可重複任務會依日期恢復每日狀態，並顯示本周已達成次數
- 任務可編輯或刪除，刪除不影響過去已累積的 EXP
- 獨立的世界探索 EXP 與世界等級
- 世界探索等級已整理為 Lv.1 到 Lv.10 的稱號、介紹、Buff、道具
- 世界探索升等後解鎖徽章、稱號、Buff、道具
- 不同技能會顯示不同顏色的標籤
- 背包介面將 Buff 和道具分開顯示
- 可自行新增任務
- 可自行新增主線副本
- 主線副本預設為空，可自行新增與刪除
- 主線副本可設定總等分與目前進度
- 副本進度會顯示百分比進度條
- 點擊副本名稱可切換任務詳情
- 手機底部導覽列
- 可加入手機主畫面
- 可匯出 / 匯入本機存檔

## GitHub Pages 上傳時請包含

請將資料夾內的檔案放到 GitHub repository 的根目錄。

repository 內應該直接看到：

- `index.html`
- `app.js`
- `styles.css`
- `manifest.webmanifest`
- `sw.js`
- `.nojekyll`
- `icons/app-icon.svg`
- `README.md`
- `人生養成遊戲_部署說明.md`

請不要把整個資料夾包在 repository 裡。也就是說，建議不要變成：

`life-rpg-pwa-zeabur/index.html`

而是要讓 `index.html` 直接位於 repository 第一層。

## GitHub Pages 設定

上傳檔案後，到 repository 的：

`Settings → Pages`

設定：

- Source：`Deploy from a branch`
- Branch：`main`
- Folder：`/root`

儲存後等待 GitHub Pages 發布完成。

發布成功後，網站網址通常會像這樣：

`https://你的帳號.github.io/你的repository名稱/`

## 手機使用方式

1. 用手機瀏覽器打開 GitHub Pages 網址。
2. 確認網站可以正常顯示。
3. 使用瀏覽器的「加入主畫面」功能。
4. 之後可以從手機主畫面的圖示開啟。

## 資料保存提醒

這版不需要資料庫。打卡資料會保存在使用者目前瀏覽器的 localStorage。

請注意：

- 同一個手機、同一個瀏覽器、同一個網址，資料通常會保留。
- 換手機、換瀏覽器、換網址時，資料不會自動同步。
- 不要隨意修改 `app.js` 裡的 `KEY`。
- 不要在沒有備份的情況下按「重置封測資料」。
- 大幅修改網站前，建議先到「報告」頁匯出存檔。

若資料遺失或換裝置使用，可以到「報告」頁使用「匯入存檔」恢復。
